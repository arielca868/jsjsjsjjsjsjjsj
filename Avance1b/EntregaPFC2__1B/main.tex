\section{Introducción}
% =====================================================================

\subsection{Propósito del documento}
El presente documento constituye la Especificación de Requisitos de Software (ERS/SRS) del Sistema para la Gestión Inteligente de una Camaronera, denominado \textbf{AquaGest}, elaborado conforme al estándar IEEE/ISO/IEC 29148:2018~\cite{ieee29148} en el marco de la asignatura \textit{Ingeniería de Requerimientos} (4to nivel) de la Carrera de Ingeniería de Software de la Universidad Técnica Estatal de Quevedo (UTEQ).

El documento está dirigido a los siguientes interesados:

\begin{itemize}
    \item \textbf{Desarrolladores de software:} como especificación técnica de las funcionalidades que deberá implementar el sistema.
    \item \textbf{Equipo de pruebas (testers):} como base para el diseño de casos de prueba y criterios de aceptación.
    \item \textbf{Stakeholders del cliente (Compañía Biofina):} para validar que el sistema satisfaga sus necesidades operativas y de negocio.
    \item \textbf{Docente evaluador:} para la revisión académica del proceso de ingeniería de requisitos aplicado.
\end{itemize}

La Entrega 1A cubrió la fase de planificación y elicitación. La presente Entrega 2 (1B) acumula y reemplaza a la Entrega 1A, incorpora las correcciones señaladas por el docente y añade las secciones correspondientes a los requisitos específicos formalizados (RF y NFR), interfaces externas mediante mockups, modelado UML, priorización MoSCoW y la matriz de trazabilidad parcial.

\subsection{Alcance del producto}

\textbf{Nombre del software:} AquaGest --- Sistema para la Gestión Inteligente de Camaroneras.

\vspace{0.3cm}

\textbf{¿Qué hará el sistema?}

\begin{itemize}
    \item Gestionar el registro y seguimiento de estanques/piscinas (identificación, estado, capacidad de siembra).
    \item Controlar el ciclo completo de siembra: fechas, cantidad de larvas, proveedor y responsable.
    \item Registrar y monitorear parámetros del agua (oxígeno, pH, temperatura, salinidad, turbidez, alcalinidad) con alertas automáticas ante valores fuera de rango.
    \item Gestionar el plan de alimentación diario por estanque, ajuste de raciones y registro de consumo.
    \item Administrar el historial sanitario: detección de enfermedades, tratamientos y seguimiento.
    \item Gestionar empleados, sus roles y la asignación de tareas diarias.
    \item Controlar inventario de bodega (balanceado, medicinas, herramientas, repuestos, combustible).
    \item Registrar cosechas y calcular rendimiento por estanque/hectárea.
    \item Generar reportes de producción, costos y análisis comparativos entre ciclos.
    \item Incorporar componentes de Inteligencia Artificial para predicción de cosecha óptima, detección de anomalías en parámetros de agua y recomendaciones de alimentación~\cite{ref2}.
\end{itemize}

\textbf{¿Qué NO hará el sistema (exclusiones explícitas)?}

\begin{itemize}
    \item No gestionará la logística de transporte hacia mercados externos.
    \item No incluirá módulos de contabilidad general ni facturación.
    \item No controlará sistemas de riego externos a la camaronera.
    \item No ofrecerá integración con plataformas de comercio electrónico.
\end{itemize}

\textbf{Beneficios esperados:}

\begin{itemize}
    \item Reducción de pérdidas por mortalidad no detectada oportunamente.
    \item Optimización del uso de alimento balanceado.
    \item Trazabilidad completa de cada ciclo productivo.
    \item Toma de decisiones basada en datos e inteligencia artificial.
\end{itemize}

\subsection{Perspectiva del producto}
AquaGest es un sistema de información nuevo, diseñado para reemplazar los procesos manuales actuales de la Compañía Biofina (Camaronera El Guasmo). Actualmente, el registro de actividades se realiza en hojas de papel y cuadernos físicos, sin trazabilidad digital ni capacidad analítica [2].
El sistema operará como una aplicación web accesible desde navegadores de escritorio y dispositivos móviles. Se integrará con sensores IoT para la captura automática de parámetros del agua [3], y contará con un módulo de IA para análisis predictivo.

\subsection{Funciones del producto}
A continuación un resumen de alto nivel de las funciones principales de AquaGest:

\begin{itemize}
    \item \textbf{Gestión de Estanques:} registro, consulta y actualización del estado de cada piscina (activa, en preparación, en cosecha, inactiva).
    
    \item \textbf{Control de Siembras:} registro de cada ciclo productivo con información de larvas, proveedor y seguimiento semanal de grameo.
    
    \item \textbf{Monitoreo de Parámetros del Agua:} visualización en tiempo real y alertas automáticas cuando los valores salen del rango óptimo.
    
    \item \textbf{Gestión de Alimentación:} cálculo y registro diario de raciones por estanque, ajuste según biomasa estimada.
    
    \item \textbf{Historial Sanitario:} registro de incidentes, diagnósticos y tratamientos aplicados.
    
    \item \textbf{Gestión de Empleados y Tareas:} administración de roles, asignación de actividades diarias y control de cumplimiento.
    
    \item \textbf{Inventario de Bodega:} control de stock de insumos (balanceado, medicinas, repuestos, combustible).
    
    \item \textbf{Gestión de Cosechas:} registro de rendimiento, peso total y talla del camarón por estanque.
    
    \item \textbf{Reportes y Análisis:} generación de reportes de producción, costos y comparativos entre ciclos.
    
    \item \textbf{Módulo de Inteligencia Artificial:} predicción de momento óptimo de cosecha, detección de anomalías en calidad del agua y recomendaciones de alimentación.
\end{itemize}

\subsection{Entorno operativo}
\textbf{Software:} el sistema deberá ser una aplicación web compatible con los navegadores Chrome (v100+) y Firefox (v100+). La base de datos deberá ser relacional (PostgreSQL o MySQL).

\textbf{Conectividad:} las instalaciones de la camaronera cuentan con conectividad a internet limitada; el sistema deberá soportar operación offline con sincronización posterior para los módulos críticos (registro de alimentación y parámetros de agua).

\textbf{Hardware:} los dispositivos de usuario serán principalmente tabletas Android y computadoras de escritorio de gama media. El sistema no podrá requerir hardware especializado en el lado del cliente.

\textbf{Integración externa:} sensores IoT para parámetros del agua (pH, oxígeno, temperatura, salinidad, turbidez, alcalinidad), adquiridos de forma independiente por la empresa.

\subsection{Restricciones}

\textbf{Regulatorias:} el sistema deberá cumplir con la normativa ecuatoriana de acuicultura y las disposiciones de la Subsecretaría de Acuacultura del Ecuador.


\textbf{Seguridad:} los datos de producción son confidenciales para la empresa; se requerirá autenticación de usuarios y control de acceso basado en roles (RBAC).


\textbf{Presupuesto:} el desarrollo deberá priorizar soluciones de bajo costo (frameworks y bibliotecas de código abierto).

\subsection{Suposiciones y dependencias}
\begin{itemize}
    \item Se asume que la Compañía Biofina dispondrá de al menos un servidor local o servicio de nube para el alojamiento del sistema.
    
    \item Se asume que los usuarios recibirán capacitación básica para el uso del sistema antes de su despliegue.
    
    \item Se asume que los sensores IoT para parámetros del agua serán adquiridos por la empresa de forma independiente al desarrollo del software.
    
    \item Si la empresa decide cambiar de proveedor de almacenamiento en la nube, los requisitos de portabilidad de datos deberán revisarse.
    
    \item La precisión del módulo de IA dependerá de la disponibilidad de datos históricos de al menos dos ciclos productivos completos.
\end{itemize}
